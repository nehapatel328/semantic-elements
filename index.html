Semantic Elements

Semantic elements are HTML5 elements that define types of content that compose a web page, both to the website browser and developer. In other words, a semantic element will define the function of a line of text within a webpage. For example, if you want a form, you would use the element <form> , which gives a clear command. In the past, vague elements, such as <span> , were used in conjunction with IDâ€™s and classes. However, a semantic element can be clearly recognized by the browser and any developer. When properly organized, semantic elements can work together to intuitively structure the information hierarch of a webpage. You can begin with a <header> element that will let the browser, developer, and even reader know what the content of the page will be about to the actual <article> with the content and end with a <footer> that has relevant copyright information.

Use semantic elements too:

  1. Define content
  2. Organize content
  3. Identify content
